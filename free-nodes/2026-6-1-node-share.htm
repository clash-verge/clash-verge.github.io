全面掌握Quantumult配置文件：安装、编写与实用指南

一、Quantumult简介：什么是Quantumult？

Quantumult是由QUANTUM核心团队开发的一款专业代理工具，主要面向iOS系统用户。它支持HTTP、SOCKS5等多种代理协议，能够帮助用户绕过网络限制，访问被屏蔽的网站和服务。Quantumult支持高度自定义的配置，使用户能够根据个人需求调整代理节点、流量规则和脚本功能，极大提升网络访问的自由度和体验。

Quantumult的核心功能

  • 多重代理模式
    用户可灵活添加并管理多条代理服务器线路，满足不同访问需求和不同网络环境下的切换。

  • 灵活规则配置
    通过域名、关键词等规则判断流量走向，实现部分流量直连，部分流量走代理，节省资源同时提升速度。

  • 节点测速
    内置测速功能，帮助用户快速判断各代理节点的实时性能，方便选择最快的节点。

  • 脚本支持
    支持JavaScript脚本，扩展更多自动化功能，如节点自动更新、测速自动执行等。

三、Quantumult配置文件详解

Quantumult的配置文件通常以quantumult.conf为扩展名，是一份文本文件，包含多个功能模块，主要分为以下几部分：

3.1 节点配置

节点即代理服务器的详细信息，包括服务器地址、端口和协议类型，是Quantumult运行的基础。

示例：

ini
[Geometry]
name = Example US Server
type = http
hostname = us.example.com
port = 8080

  • name：节点名称，用于识别和选择。

  • type：协议类型，如http、socks5。

  • hostname：服务器域名或IP地址。

  • port：服务端口。

3.2 规则配置

规则定义了不同流量如何处理，常见有“直接连接”与“代理连接”两种策略。

示例：

pgsql
[Rule]
DOMAIN-SUFFIX, example.com, Proxy
DOMAIN, example.net, Direct

  • 通过规则，指定访问example.com的流量走代理服务器。

  • 访问example.net则直接连接，不经过代理。

这类规则使用户可以灵活控制网络访问路径，提高访问效率，减少不必要的代理资源浪费。

3.3 脚本设置

Quantumult支持用户编写或导入JavaScript脚本，来扩展功能，比如节点自动切换、节点测速、自动更新配置等。

示例：

csharp
[Script]
// 此处添加用户自定义脚本代码

脚本部分提升了Quantumult的自动化能力，使网络管理更为智能。

5.4 配置错误怎么办？

错误配置会导致Quantumult无法正常连接网络。建议检查配置语法是否正确，节点信息是否填写准确。如无法恢复，建议还原备份文件或重新生成配置。

FAQ

Trojan 的伪装域名有什么要求？
Trojan 的伪装域名必须是可以通过 HTTPS 正常访问的真实网站，并且 SSL 证书有效。伪装域名用于与服务器证书匹配，使得流量在外观上完全与正常 HTTPS 请求一致。
Mihomo 节点分组如何自动切换？
用户可按延迟、地区或用途将节点分组，并启用策略组。系统会根据节点健康状况和延迟自动切换到最佳节点，保证网页浏览、视频播放和游戏连接稳定顺畅。
何为 fake-ip 模式，它的优缺点是什么？
fake-ip 是通过伪造 DNS 返回的 IP 地址让客户端透明代理流量，优点是兼容性好、无需修改应用代理设置；缺点是可能导致某些证书校验或 SNI 伪装失效，需要配合 DNS 与证书策略谨慎使用。
QuantumultX 的重写规则能做什么？
重写（Rewrite）可用于修改请求头、屏蔽广告、替换响应内容等。通过正则匹配目标请求可实现页面定制与功能增强，但应谨慎使用以避免影响正常网站加载与安全验证。
Trojan-Go 的端口不可达怎么办？
检查服务器防火墙是否开放对应端口，同时确认客户端和服务器端端口一致。还可以尝试更换节点或端口，确保 TLS 证书有效以保证连接成功。
如何配置 Clash 以支持局域网内设备共享代理？
启用 Clash 的 HTTP/SOCKS5 本地监听并在局域网设备上设置该代理地址为网关；若使用 TUN 模式需配合路由器或透明代理实现全局共享，注意开启访问控制避免未经授权的外部访问。
Sing-box 可以替换 Clash 配置文件吗？
Sing-box 支持直接解析 Clash YAML 配置，并转换为自身 JSON 格式。节点、策略组和规则基本兼容，但部分自定义脚本可能需要手动调整才能完全使用。
Surfboard 在 iOS 上无法连接时如何调试？
检查 iOS 的 VPN 配置权限是否授予，确认订阅链接或手动配置填写无误；查看 Surfboard 的日志或诊断页面，查看是否为 DNS 问题、TLS 错误或节点超时，并尝试切换节点或重启应用。
Nekoray 的订阅更新机制如何运作？
Nekoray 可以通过远程订阅 URL 自动下载最新节点和规则文件。用户可以设置更新频率，程序会定时检测并替换旧节点，保持配置最新状态。
Matsuri 节点延迟排序功能如何使用？
Matsuri 会定期测速节点延迟，并根据结果排序。用户可选择延迟最低的节点进行访问，结合策略组实现自动切换，提高网络速度和连接稳定性。
